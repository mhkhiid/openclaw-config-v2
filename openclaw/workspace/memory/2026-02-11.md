# 2026-02-11 - 诊断日志

## OpenRouter "no output" 问题诊断

### 问题现象
- 用户报告收到 "no output" 回复
- 日志显示 `FailoverError: No available auth profile for openrouter`
- 随后出现 `billing error — API key has run out of credits`

### 根本原因
**OpenRouter auth profile 配置不完整**：
```json
"openrouter:default": {
  "provider": "openrouter",
  "mode": "api_key"
  // ❌ 缺少 "apiKey" 字段！
}
```

### 同样问题
- `minimax-portal:default` OAuth 也显示 "No available auth profile"
- 可能是 OAuth token 过期

### 当前 fallback 顺序
当前可用的是 `kimi-coding/k2p5`，其他 provider 都不可用：
1. ❌ minimax-portal/MiniMax-M2.1 (OAuth 问题)
2. ❌ openrouter/* (API Key 缺失)
3. ✅ kimi-coding/k2p5 (正常工作)

### 修复方法
需要运行 `openclaw configure` 重新配置 OpenRouter API Key，或手动编辑 `~/.openclaw/openclaw.json` 添加 apiKey 字段。
